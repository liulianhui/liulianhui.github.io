nohup进程后台运行、查看、终止

发表于 2018-03-05 | 分类于 centos ， python
简述

关于nohup和&，进程后台运行、查看、终止

nohup和&

nohup 命令可以使命令永久的执行下去，和终端没有关系，退出终端也不会影响程序的运行；
& 是后台运行的意思，但当用户退出的时候，命令自动也跟着退出。
那么，把两个结合起来nohup 命令 &这样就能使命令永久的在后台执行

在终端执行
nohup 命令 > output.log 2>&1 &
nohup+最后面的&是让命令在后台执行

>output.log是将信息输出到output.log日志中

2>&1是将标准错误信息转变成标准输出，这样就可以将错误信息输出到output.log 日志里面来。

查看日志

tail -f output.log（动态显示）

cat output.log（一次性显示整个文件）

查看当前Python进程

ps -ef |grep python

杀死进程

sudo kill 进程号

kill 9 进程号 #绝杀
